# Map Domain Classes

このディレクトリには、マップ機能のドメインモデルを定義する 3 つのクラスがあります。

## Room

教室や施設の詳細情報を表すドメインモデル。

### 主な機能

- **施設情報の管理**: 階数、部屋名、教室番号、ヘッダー、詳細説明、メールアドレス、検索ワードリストを保持
- **スケジュール管理**: `RoomSchedule`のリストを保持し、特定の日付で授業スケジュールをフィルタリング可能（`getScheduleListByDate`メソッド）
- **Firebase 連携**: Firebase から取得したデータをパースして`Room`インスタンスを生成（`fromFirebase`ファクトリコンストラクタ）

### 主なプロパティ

- `floor`: 階数
- `roomName`: 部屋名
- `classroomNo`: 教室番号（nullable）
- `header`: ヘッダー情報
- `detail`: 詳細説明（nullable）
- `mail`: メールアドレス（nullable）
- `searchWordList`: 検索用キーワードリスト（nullable）
- `scheduleList`: 授業スケジュールリスト（nullable）

## MapDetailMap

`Room`の集合を管理し、検索機能を提供するコンテナクラス。

### 主な機能

- **単一検索**: `searchOnce`メソッドで階数と部屋名を指定して特定の`Room`を取得
- **全体検索**: `searchAll`メソッドで検索テキストに基づいて関連する`Room`を優先度順に取得
  - 優先度 1: 部屋名が完全一致
  - 優先度 2: 検索ワードリストに部分一致
  - 優先度 3: ヘッダーまたはメールアドレスに部分一致
  - 優先度 4: 詳細説明に部分一致

### データ構造

内部的に`Map<String, Map<String, Room>>`の形式で、階数をキー、部屋名をキーとした二重マップで`Room`を管理。

## MapTileType

マップ上に表示されるタイル（部屋や施設）の種類を定義する列挙型。

### タイプ一覧

- `classroom`: 講義室（背景: ダークグレー、テキスト: 白）
- `teacherroom`: 教員室と実験室（背景: グレー、テキスト: 白）
- `subroom`: 補助的な部屋（背景: グレー、テキスト: 黒）
- `otherroom`: 倉庫など（背景: ライトグレー、テキスト: 黒）
- `wc`: トイレ（背景: グリーン、テキスト: 黒）
- `stair`: 階段（背景: ライトグレー、テキスト: 黒）
- `ev`: エレベーター（背景: ダークグレー、テキスト: 白）
- `road`: 通路（背景: ライトグレー、テキスト: 黒）
- `empty`: 吹き抜けなど（背景: 透明、テキスト: 黒）

各タイプには`backgroundColor`と`textColor`が関連付けられており、マップ UI での視覚的な区別に使用されます。
